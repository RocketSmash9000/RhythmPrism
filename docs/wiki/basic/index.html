Basic editor use

In this guide you will learn how to import, export and open a Godot project, as well as some basic knowledge about where everything is. This guide assumes you have no knowledge of how to use Godot. If you have experience or you think you can figure out things yourself, you may skip this guide.

You can download Godot from the official website. Please note that it comes as a standalone app. This means that you will not need to install anything. If you double click the executable file (godot-nonsense-win64.exe), it will open the project menu. Don't open nor delete the file that has "console" in the end. It's important and it not being there may cause problems.

You can place the executables the folder of your choice. It's recommended that you create a shortcut to the aforementioned file and place it on the desktop for easy access. It's your choice, though.

Godot for MacOS may require installation, as it's a more strict OS. I don't own a Mac, so I unfortunately can't test if it works straight out of the box or if it needs installation.

First, you will need to open Godot. In the top bar menu, you will see a few buttons. Press the one that says "Import". It will open a window for you to select where the project is located. Navigate to the folder where the project is located, and select the "project.godot" file within it. Next, press the "Open" button located at the bottom of the window.

If you did it right, Godot should show the project in the main screen.

You can open projects by double clicking on the project. You can also click on the project once and then click on the "Edit" button.

Navigate to the folder that contains the whole project. There is no built-in way to export a project, since everything is mostly plain text files in different formats. However, you can compress the project folder and share that with someone else. They can then decompress the zip file and import the project.

Godot's UI can be broken into a few key components, which are listed below.

Left panes

These two panes contain the nodes inside a given scene (including their hierarchy), and a view of all the folders inside the project.

This image is an example of how the left pane looks like. Please note that its appearance may vary depending on the theme the editor has, but its functionality and structure remains the same. 

Node hierarchy is easy to understand. There's what's called a "parent node", which is the root of the scene. Think of it as the conductor of an orchestra. All other nodes depend on the parent to be able to function, and there can only be one. Children nodes are nodes that are attached in some way to another node. For example, the "Static elements" node is a child node of the parent node. Child nodes can also have children nodes.

The file view in the bottom panel lets you explore the contents of the project without having to use an external file explorer. Godot treats the folder in which the project is in as special, so instead of having a normal name, it's called "res://". You can see it as the parent node for files.

Right panes

These panes are what lets you alter different parameters of the nodes.

They will generally look like this, although the content may vary depending on what node is selected, as each node has its functionality. There is more functionality to this pane which we won't talk about because it goes out of the scopes of this guide.

Top bar

As its name implies, this bar sits at the top of the app. It has a few utilities. The buttons we will use the most are the "2D", "Script" and the play button next to "AssetLib". "2D" switches to the 2D editor view of the project. You can see and edit what the user would see in the screen. The "Script" button switches your view to a text editor where you can write and modify code. The play button will play the project in a new window, where you can playtest.

Bottom bar

You will probably not need to use this bar much, except for opening a few menus like the SpriteFrames editor for polo animations. The Output menu is useful for debugging, if there's something that doesn't work as intended. If there's a menu open, press again on the button to close it.

Main view

The center of the UI, where you can edit scripts, visualize scenes, change node positions and so much more. No image is included here, as it normally takes up a whole lot of screen space, and would therefore take up a big portion of the screen.
